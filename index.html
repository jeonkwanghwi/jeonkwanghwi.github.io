    Project 1 : AI 구술 자서전

    "편리한 디지털 문화"를 목표로, 노인 자서전 프로젝트를 진행한 경험이 있습니다. 생성형 AI가 사용자의 자서전 작성을 돕기 위해 질문을 생성하고, 사용자가 답변한 내용을 모아 자서전을 완성하도록 구현하였습니다. 이때 사용자를 고려하여 STT와 TTS 기능을 추가하였습니다.

    (1) 타자 입력이 불편할 수 있는 사용자를 위해 Speech To Text 기능을 도입해, 음성 기반 대화 방식으로 자서전을 작성할 수 있도록 하였습니다.
    (2) 노인분들이 쉽게 이해할 수 있도록 직관적인 UI와 한글 중심의 인터페이스를 제공하였고, 글씨가 작게 느껴질 것을 대비해 Text To Sppech 기능을 추가하여 음성으로 질문을 들려주도록 구현하였습니다.

    랭체인과 프롬프트 엔지니어링을 적용하여 노인 맞춤형 AI를 개발하였습니다. 사용자의 답변을 저장하여 질문 생성 시 맥락을 이해하고 일관된 질문을 제공할 수 있도록 설계했습니다. 이때, 자서전 파일이 길어질수록 질문이 맥락을 놓치는 문제가 발생했습니다. 이를 개선하기 위해 중요 키워드만 전달해보았으나 오히려 더 맥락을 놓치게 되었습니다. 따라서 대화의 요약본과 최근 대화를 함께 전달하는 '대화 요약 메모리' 기능을 활용함으로써 맥락을 유지함과 동시에 질문 생성 시간을 50% 단축할 수 있었습니다.

    프로토타입이 완성된 후 경로당에서 알파 테스트를 진행하며 사용자 피드백을 수집했습니다. 이 과정에서 앱이 작동 중임을 나타내는 시각적 피드백이 부족하다는 점을 인식하고, 이를 해결하기 위해 프로그래스 바를 추가해 UX를 개선하였습니다. 이러한 개선은 사용자 중심 개발의 중요한 성과로 이어졌으며, 경로당 테스트에서도 긍정적인 반응을 얻었습니다.

    이후 ISTQB 자격증 공부를 통해 소프트웨어 품질 보증 및 테스트 케이스 설계에 대한 깊이 있는 학습을 하였으며, 이를 바탕으로 프로젝트의 품질을 더욱 향상시키는 데 기여했습니다.

    앞으로도 기술 발전이 모든 세대에게 가치를 제공할 수 있도록 노력하며, 따뜻하고 포용적인 IT 환경을 구축하기 위해 최선을 다하겠습니다.

    Project 2 : 블록체인 유언장

    할머니의 별세 이후, 그 어떤 유언도 듣지 못한 아버지에게 아무것도 해드릴 수 없어 속상했습니다. 이런 후회를 다시는 남기지 않겠다고 결심하며 온라인 유언장 작성 프로젝트를 시작하였습니다.

    초기 모델은 단순 저장 기능만 있던 유언장 작성 서비스였지만 위변조 방지가 중요했기에 블록체인으로 이를 해결하고자 했습니다. 경로당에서 노인분들과 대화하며 필요한 기능을 조사하던 중, 유언장의 법적 효력을 보장하는 "공증"을 알게 되었습니다. 변호사와 상담하며 공증에 대해 학습하였고, 5가지 공증 방법 중 온라인에서도 가능한 녹음으로 공증받고자 했습니다.

    서비스에 녹음 기능을 추가하고 녹음 파일을 NFT처럼 만들면 될 것이라고 생각했는데, 이를 위해선 대체 불가능한 ERC-721 프로토콜을 사용해야 했습니다. 이미 ERC-20으로 개발 중이었기에 이를 전환하는 것은 어렵다는 팀원의 의견이 있었습니다. 그러나 단순히 어렵다는 이유로 시도하지 않는 것은 안 된다고 생각했기에 제가 주도적으로 이끌겠다는 약속으로 팀원들을 설득할 수 있었습니다. 녹음 파일의 URL을 tokenURI로 설정하고, ERC-721의 mint 기능을 사용하여 NFT를 생성했습니다. 또한, safeTransferFrom, ownerOf 메소드를 추가하여 유언장의 소유권과 인증 기능을 구현했습니다.

    결과적으로 녹음 파일 URL을 민팅하여 NFT화함으로써 위변조 방지와 공증 기능을 모두 해결할 수 있었습니다. 이를 뒷받침하기 위해 유언장의 위변조 방지 기능을 테스트하였습니다. 파일의 변경 및 삭제, 내용 수정 등의 시도를 통해 URL이 변경되었을 때 블록체인에서 즉각적으로 감지되었고, 해시값 비교를 통해 유언장 내용의 위변조를 자동으로 확인할 수 있었습니다. 이러한 테스트를 통해 블록체인의 무결성과 보안성을 성공적으로 입증하였고, 이 성과로 SW 경진대회에서 최우수상을 받을 수 있었습니다.

    이 경험을 통해 어려움 속에서도 끝까지 해결책을 찾는 자세가 중요하다는 것을 깨닫고, 새로운 기술에 도전할 때도 두려워하지 않고 도전하는 자세를 배웠습니다.

    Project 3 : 공모전 공유 서비스

    Spring을 이용한 공모전 공유 서비스 프로젝트를 진행하며 카카오 소셜 로그인 기능을 개발한 경험이 있습니다. 이 과정에서 외부 API 연동, DB 관리, 세션 관리를 익혔습니다. 사용자가 카카오 로그인을 시도할 때 인증 코드를 받아 액세스 토큰을 생성하고, 이를 통해 사용자 정보를 조회한 후 DB에서 등록 여부를 확인하여 로그인 또는 신규 회원가입을 처리했습니다. 또한, 세션 무효화를 통해 접근을 관리하였습니다. 이러한 경험은 실무에서 외부 서비스를 사용하고 운영할 때 큰 도움이 될 것입니다.

    여러 프로젝트를 하며 책임감을 기를 수 있었습니다. "빠르게 만들고 피드백을 받자"는 철학으로 항상 마감 기한보다 앞당겨 개발하였습니다. 이러한 접근으로 서로가 생각하는 개발 방향성을 여러 번 확인할 수 있었고, 합의된 결과물을 도출해 낼 수 있었습니다.

    이를 통해 외부 API를 사용한 개발을 경험할 수 있었고, DB연동과 세션관리 등을 배울 수 있었습니다. 또한 팀 내 협업에 대해 배울 수 있었습니다. 협업 툴을 사용하는 것뿐만 아니라, 팀원들을 대하는 태도가 중요하다고 생각했습니다. 프로젝트 진행 시 문제의 상황에서 상대의 입장에서 한번 생각하며 이해하려고 노력했습니다. 이로인해 문제가 갈등으로 이어지지 않을 수 있었고 상호 존중의 분위기로 이끌 수 있었습니다.

    Project 4 : 정보 공유 커뮤니티

    Spring을 이용해 정보 공유 커뮤니티를 개발한 경험이 있습니다. Spring MVC 패턴을 적용해 개발하였고, view는 타임리프를 사용하여 구성했습니다. 개발 중 게시물 로딩 시 지연되는 현상을 발견하고 로그를 분석해 보니, 1개의 게시글 쿼리 실행 후 추가적인 N개의 쿼리가 발생하는 N+1 문제가 있었습니다. 처음에는 즉시 로딩을 적용해 해결하려 했으나 필요 없는 데이터까지 가져오는 over-fetching 문제가 발생했습니다. 이를 지연 로딩과 Fetch join을 함께 적용하여 연관된 데이터를 한 번에 조회하는 방식으로 개선하였습니다. 이를 통해 DB 조회 없이 바로 데이터를 반환할 수 있었고, 댓글이 프록시 객체가 아닌 실제 댓글 객체로 생성되어 N+1 문제를 효과적으로 해결할 수 있었습니다.

    이후 AWS Lightsail, MobaXterm, FileZilla를 활용하여 배포하였고, JAR 파일 전송을 자동화했습니다. 스크립트를 작성하여 새로 생성된 JAR 파일이 지정된 폴더로 자동 업로드되도록 설정해 수동 전송의 번거로움을 줄였습니다. 프로젝트를 통해 엔티티 구성, 문제 해결, 사용자 관리 등을 경험하며 서비스 운영의 전반적인 흐름을 배울 수 있었습니다.
