    Projects

    Project 1: AI 구술 자서전

    산학협력 프로젝트에서 FastAPI와 프롬프트 엔지니어링을 맡아 AI 구술 자서전을 개발한 경험이 있습니다. 노인분들이 핸드폰 타자를 어려워할 것을 예상하여 STT 기술을 도입하여 말로써 자서전을 쓰이도록 의도하였습니다. 이때, 노인 스스로 자서전 내용을 채울 수 있도록 생성형 AI를 이용하여 질문을 생성하였고, 사람마다 특화된 질문이 나오도록 프롬프트 엔지니어링을 도입하였습니다.

    사용자별 답변 history 정보를 DB에 저장해두었다가 자서전을 쓸 때는 프론트엔드 측에서 로그 형태로 들고 있는 방식으로 구현하였습니다. 이를 통해 질문생성 요청시 DB 조회없이 바로 GenAI 서버로 질문을 요청함으로써 프로세스를 간소화할 수 있었고, 실제 시간은 50%정도 감축할 수 있었습니다. history 정보를 GenAI가 받고, 이를 기반으로 사용자의 맥락을 유지한채 새로운 질문을 생성하게 됩니다. 사용자의 답변은 DB에 계속 history로써 저장되고, 프론트엔드 측에서도 로그를 업데이트하게 됩니다.

    1차 프로토타입이 완성되었을 때 인근 경로당에 가서 자서전 작성 봉사활동을 진행하였고, 이때 받았던 피드백을 기반으로 개선작업을 거쳤습니다.

    Project 2: 블록체인 유언장

    할머니의 별세에서 그 어떤 유언도 듣지 못한 아버지에게 아무것도 해드릴 수 없어 속상했고, 다시는 이런 후회를 남기지 않겠다고 결심하며 온라인 유언장 작성 프로젝트를 시작하였습니다.

    초기 모델은 단순한 온라인 유언장 작성 서비스였습니다. 그러나 유언장의 특성상 상속을 빼놓을 수 없었고, 이를 포함하기 위해서는 유언장의 위변조 방지가 매우 중요했습니다. 이를 위해 블록체인을 도입하였고 이때 대체 불가능한 특성이 있는 ERC-721 프로토콜을 사용함으로써 상속 내용의 고유성을 보장하고자 했습니다.

    개발하며 유언장을 법적으로 인정받는 공증이라는 요소를 알게 되었고 변호사와 상담하며 공증 방식을 학습하였습니다. 이때 온라인에서도 편리하게 이용할 수 있는 녹음을 통한 공증을 채택하였습니다. 녹음 파일 url을 DB에 저장하고, 이를 민팅함으로써 유언장의 NFT화를 이루어냈습니다. 이러한 기술적인 성과를 인정받아 교내 SW 경진대회에서 최우수상을 받을 수 있었습니다.

    프로젝트를 통해 새로운 분야에서도 노력과 끈기로 해낼 수 있다는 자신감을 얻을 수 있었습니다. 이후 외할머니, 외할아버지와 함께 유언장 작성 서비스를 체험하는 시간을 가졌습니다. 아버지께서 할머니를 떠나보내며 겪었던 후회를 어머니께는 남기고 싶지 않았기 때문입니다. 이러한 경험을 통해 단순히 기술 개발에 그치지 않고, 실제로 사람들에게 가치를 전달하는 사람이 되어야겠다고 다짐하였습니다.

    Project 3: 공모전 공유 서비스

    첫 스프링 프로젝트로 공모전 공유 서비스를 개발하였습니다. Spring MVC 구조를 학습하고, Entity와 DTO의 기본적인 구조를 이해하였습니다. 이 프로젝트에서 카카오 소셜 로그인 기능을 맡아 회원가입을 구현하였습니다.

    사용자가 카카오 로그인을 할 때 인증 코드를 받아, 이를 통해 액세스 토큰을 얻는 방식으로 구현하였습니다. 액세스 토큰으로 사용자의 정보를 조회한 후, DB에서 해당 사용자가 이미 등록되어 있는지 확인하였습니다. 이미 등록된 사용자는 로그인 처리하고, 신규 사용자는 랜덤으로 닉네임을 생성하여 회원가입을 진행하였습니다. 로그인 성공 시 사용자 정보를 반환하고, 로그아웃 시에는 세션을 무효화하여 보안 처리를 하였습니다.

    이를 통해 외부 API를 사용한 개발을 경험할 수 있었고, DB연동과 세션관리 등을 배울 수 있었습니다.
