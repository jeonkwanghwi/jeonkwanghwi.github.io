# 따뜻한 엔지니어

노년층의 삶의 질 개선에 관심이 많아 관련 프로젝트를 다수 진행하였습니다. (1) 재가노인 우울감 개선 프로젝트, (2) AI 구술 자서전, (3) 블록체인 유언장, (4) 독거노인 스마트 하우스 프로젝트를 진행한 바 있습니다. 이러한 프로젝트로 노인분들이 삶을 돌아보며 인생에서 새로운 가치를 찾기를 원했습니다. 프로젝트에서는 사용자 중심 개발을 가장 중요하게 생각했고, 그들의 불편함을 줄이는 데 최선을 다했습니다.
현재는 사회공헌 창업동아리 "인액터스"에서 재가 노인을 대상으로 심리치료 프로젝트를 진행하고 있습니다. 앞으로도 사회에 공헌할 수 있는 프로젝트를 계속할 것이고, 따뜻한 세상을 만들어가겠습니다.

# 커뮤니케이션

학부 시절, 15개 이상의 프로젝트에 참여하며 의사소통 능력을 길러왔습니다. 한 프로젝트에서는 팀원들이 요구사항 명세서를 각기 다르게 이해하면서 개발이 지연되고, 결국 갈등으로 이어진 적이 있었습니다.
문제를 해결하기 위해 각 팀원과 개별적으로 만나 상황을 파악하고, 의견을 수렴했습니다. 이때 공통으로 느낀 문제의 원인은 "명세서의 모호함"이었습니다. 이후 팀원들을 한자리에 모아 대화를 제안했고, 오해를 풀고 분위기를 회복하였습니다.
프로젝트가 원활하게 재개되도록 각자의 역할을 정리해 팀원들에게 전달했고, 동일한 문제가 재발하지 않도록 요구사항 명세서를 명확하게 수정했습니다. 또한, 매일 10분씩 짧은 회의를 열어 업무 상황을 공유하며 팀의 진행 상황을 지속적으로 조율했습니다.
그 결과, 개발이 지연되었음에도 불구하고 최종 마감 기한을 20% 앞당길 수 있었고, 팀원 모두가 A학점을 받는 성과를 거두었습니다. 이 경험을 통해 대화와 이해심이 팀의 성공에 있어 얼마나 중요한 요소인지 다시 한번 깨달았습니다.

제 장점은 습관적인 메모입니다. 일정 관리, 궁금한 점, 새롭게 알게 된 것을 반드시 핸드폰, 메모장, 노트북 등 모든 곳에 메모해 놓습니다. 그리고 이를 블로그 에 정리하며 내용을 정리하고 배워가고 있습니다. 현재까지 330개가 넘는 글을 작성했으며, CS, 알고리즘, 프로젝트 회고, 생각 정리 등 다양한 주제를 다루고 있습니다. 이러한 습관으로 작은 것이라도 기억하며 성장할 수 있었습니다.

제 단점은 업무를 할 때 종종 과몰입하는 경향이 있다는 것입니다. 한때는 과정보다 성과에 더 집중했던 적이 있었습니다. 그러다 보니 팀원들의 실수나 시간 약속 등에서 예민하게 반응했고, 결과적으로 만족하지 못했습니다. 이에 대해 팀원들과 진지하게 이야기를 해봤고, "성과도 중요하지만, 과정을 즐기면 결과는 따라온다"는 것을 깨달았습니다. 그 후로는 작은 것에 집착하지 않고 팀원들과 함께하는 시간을 즐기며 소통하려 노력했습니다. 이제는 모임 자체를 즐기고 있고, 이전보다 과정에 중점을 두며 프로젝트를 진행하고 있습니다. 성과도 중요하지만, 팀원들과의 관계 형성과 이해심이 더 중요하다는 것을 깨달은 계기였습니다.

Project 1 : AI 구술 자서전

"편리한 디지털 문화"를 목표로, 노인 자서전 프로젝트를 진행한 경험이 있습니다. 생성형 AI가 사용자의 자서전 작성을 돕기 위해 질문을 생성하고, 사용자가 답변한 내용을 모아 자서전을 완성하도록 구현하였습니다. 이때 사용자를 고려하여 STT 기능을 통해 음성 기반 대화 방식으로 자서전을 작성할 수 있도록 하였습니다.

랭체인과 프롬프트 엔지니어링을 적용하여 노인 맞춤형 AI를 개발하였습니다. 사용자의 답변을 저장하여 질문 생성 시 맥락을 이해하고 일관된 질문을 제공할 수 있도록 설계했습니다. 이때, 자서전 파일이 길어질수록 질문이 맥락을 놓치는 문제가 발생했습니다. 이를 개선하기 위해 중요 키워드만 전달해보았으나 오히려 더 맥락을 놓치게 되었습니다. 따라서 대화 요약을 통해 맥락을 파악하는 ConversationSummaryMemory를 사용하여 맥락을 관리하였습니다. 결과적으로 질문 생성 시간을 50% 단축할 수 있었습니다.

프로토타입이 완성된 후 경로당에서 알파 테스트를 진행하며 사용자 피드백을 수집했습니다. 이 과정에서 앱이 작동 중임을 나타내는 시각적 피드백이 부족하다는 점을 인식하고, 이를 해결하기 위해 프로그래스 바를 추가해 UX를 개선하였습니다. 뿐만 아니라, 글씨가 잘 안보이시는 것을 인지하고 TTS 기능을 추가하였고, 앱의 사용 메뉴얼을 더 직관적으로 개선하였습니다. 이러한 사용자 중심의 개발을 통해 품질을 더 높일 수 있었고, 그 결과 분반 1등이라는 성과를 거둘 수 있었습니다.

앞으로도 기술 발전이 모든 세대에게 가치를 제공할 수 있도록 노력하며, 친절한 IT 환경을 구축하기 위해 최선을 다하겠습니다.

Project 2 : 블록체인 유언장

할머니의 별세 이후, 그 어떤 유언도 듣지 못한 아버지에게 아무것도 해드릴 수 없어 속상했습니다. 이런 후회를 다시는 남기지 않겠다고 결심하며 온라인 유언장 작성 프로젝트를 시작하였습니다.

초기 모델은 단순 저장 기능만 있던 유언장 작성 서비스였지만 위변조 방지가 중요했기에 블록체인으로 이를 해결하고자 했습니다. 경로당에서 노인분들과 대화하며 필요한 기능을 조사하던 중, 유언장의 법적 효력을 보장하는 공증을 알게 되었습니다. 변호사와 상담하며 5가지 공증 방법 중 온라인에서도 가능한 녹음으로 공증받고자 했습니다.

서비스에 녹음 기능을 추가하고 녹음 파일을 NFT 처럼 만들면 될 것이라고 생각했는데, 이를 위해선 대체 불가능한 ERC-721 프로토콜을 사용해야 했습니다. 이미 ERC-20 으로 개발 중이었기에 이를 전환하는 것은 어렵다는 팀원의 의견이 있었습니다. 그러나 단순히 어렵다는 이유로 시도하지 않는 것은 안 된다고 생각했기에 제가 주도적으로 이끌겠다는 약속으로 팀원들을 설득할 수 있었습니다. 녹음 파일의 URL을 tokenURI로 설정하고, ERC-721의 mint 기능을 사용하여 NFT를 생성했습니다.

결과적으로 녹음 파일 URL을 minting하여 NFT화함으로써 위변조 방지와 공증 기능을 모두 해결할 수 있었습니다. 이를 뒷받침하기 위해 유언장의 위변조 방지 기능을 테스트하였습니다. 파일의 변경 및 삭제, 내용 수정 등의 시도를 통해 URL이 변경되었을 때 블록체인에서 즉각적으로 감지되었고, 해시값 비교를 통해 유언장 내용의 위변조를 자동으로 확인할 수 있었습니다. 이러한 테스트를 통해 블록체인의 무결성과 보안성을 성공적으로 입증하였고, 이 성과로 SW 경진대회에서 최우수상을 받을 수 있었습니다.

이 경험을 통해 어려움 속에서도 끝까지 해결책을 찾는 자세가 중요하다는 것을 깨닫고, 새로운 기술에 도전할 때도 두려워하지 않고 도전하는 자세를 배웠습니다.

Project 3 : 공모전 공유 서비스

Spring을 이용한 공모전 공유 서비스 프로젝트에서 카카오 소셜 로그인 기능을 개발하며 외부 API 연동, DB관리, 세션 관리를 구현한 경험이 있습니다.

카카오 로그인을 구현할 때, 사용자 인증 과정을 두 단계로 나누어 진행했습니다. 먼저 사용자가 로그인을 시도하면, 인증 코드를 기반으로 액세스 토큰을 발급하였고, 이를 이용해 카카오 서버에서 사용자 정보를 조회했습니다. 조회한 정보를 바탕으로 해당 사용자가 이미 우리 DB에 존재하는지 확인하고, 기존 회원일 경우는 로그인을 허용, 신규 회원일 경우 회원가입 페이지로 redirect하여 절차를 이어갔습니다.

이 프로젝트에서 저는 배워가는 입장이었기 때문에 "빠르게 만들고 피드백을 통해 개선하자"는 철학을 가지고 임했습니다. 항상 기한보다 앞서 개발을 완료하여 개발 방향성을 여러 차례 피드백받았고, 이를 통해 최종적으로 합의된 결과물을 도출하는 데 주력했습니다. 이 경험을 통해 외부 API 사용, DB 연동, 세션 관리 등 백엔드의 핵심 요소들을 학습할 수 있었으며, 주어진 환경에서 최대한 효율적으로 일하는 방법을 익혔습니다. 앞으로도 배움을 지속하며, 더 나은 결과를 위해 끊임없이 성장하는 엔지니어가 되겠습니다.

Project 4 : 정보 공유 커뮤니티

Spring을 이용해 정보 공유 커뮤니티를 개발한 경험이 있습니다. Spring MVC 패턴을 적용해 개발하였고, view는 타임리프를 사용하여 구성했습니다. 개발 중 게시물 로딩 시 지연되는 현상을 발견하고 로그를 분석해 보니, 1개의 게시글 쿼리 실행 후 추가적인 N개의 쿼리가 발생하는 N+1 문제 가 있었습니다. 처음에는 즉시 로딩을 적용해 해결하려 했으나 필요 없는 데이터까지 가져오는 over-fetching 문제가 발생했습니다. 이를 지연 로딩과 Fetch join을 함께 적용하여 연관된 데이터를 한 번에 조회하는 방식으로 개선하였습니다. 이를 통해 DB 조회 없이 바로 데이터를 반환할 수 있었고, 댓글이 프록시 객체가 아닌 실제 댓글 객체로 생성되어 N+1 문제를 효과적으로 해결할 수 있었습니다.

이후 AWS Lightsail, MobaXterm, FileZilla를 활용하여 배포하였고, JAR 파일 전송을 자동화했습니다. 스크립트를 작성하여 새로 생성된 JAR 파일이 지정된 폴더로 자동 업로드되도록 설정해 수동 전송의 번거로움을 줄였습니다. 프로젝트를 통해 엔티티 구성, 문제 해결, 사용자 관리 등을 경험하며 서비스 운영의 전반적인 흐름을 배울 수 있었습니다.
